# 前端源码分析

## 1. 模块概述

前端位于 `apps/desktop/`，是一个 **Vue 3 + TypeScript** 单页应用，同时运行在 Tauri WebView（桌面版）和浏览器（Web 版）中。通过**双后端适配层**（Tauri IPC / HTTP REST）实现一套前端代码对接两种运行模式。

| 属性 | 值 |
|------|-----|
| **框架** | Vue 3.5（Composition API + `<script setup>`） |
| **构建工具** | Vite 8 |
| **状态管理** | Pinia 3 |
| **SQL 编辑器** | CodeMirror 6 |
| **UI 组件库** | shadcn-vue（reka-ui）+ Tailwind CSS 4 |
| **国际化** | vue-i18n（7 种语言） |
| **图表** | ECharts 6 + vue-echarts |
| **源码文件** | ~830 个（`.vue` + `.ts`） |
| **总代码量** | ~120,000+ 行 |
| **路径别名** | `@/` → `src/` |

## 4. 入口与启动流程（main.ts）

```typescript
async function bootstrap() {
  // 1. 并行加载 i18n 模块 + App.vue（动态导入）
  const [{ default: i18n, loadSavedLocale }, { default: App }] =
    await Promise.all([import("./i18n"), import("./App.vue")]);

  // 2. 加载语言包
  await loadSavedLocale();

  // 3. 创建 Vue 应用
  const app = createApp(App);
  app.use(createPinia());        // Pinia 状态管理
  app.use(i18n);                 // 国际化
  app.use(VueVirtualScroller);   // 虚拟滚动
  app.mount("#root");

  // 4. 全局 input 属性注入（禁用 autocomplete/autocapitalize/spellcheck）
  installGlobalInputAttrs();
}
```

**启动保护**：
- `installStartupErrorHandlers()` — 捕获全局 `error` / `unhandledrejection`
- `installDebugLogCapture()` — 捕获调试日志
- `renderStartupError()` — 启动失败时渲染错误卡片（含堆栈信息）

## 9. 核心组件

### 9.1 数据网格（DataGrid.vue, 12343行）

前端**最大的单文件组件**，实现类似 Excel 的数据浏览/编辑：

- **虚拟滚动**：vue-virtual-scroller，支持百万行数据
- **单元格编辑**：文本/数字/日期/枚举/JSON/空间数据
- **行内编辑**：直接修改 → 生成 UPDATE SQL → 批量保存
- **列操作**：排序/过滤/隐藏/调整宽度/冻结
- **行选择**：单选/多选/全选/范围选
- **复制粘贴**：CSV/JSON/Markdown 格式
- **分页**：服务端分页 + 无限滚动
- **导出**：CSV/XLSX/JSON/Markdown/SQL INSERT
- **条件格式**：基于值的单元格高亮
- **转置视图**：行列互换显示

### 9.2 SQL 编辑器（QueryEditor.vue, 3685行）

基于 CodeMirror 6 的 SQL 编辑器：

- **SQL 方言**：根据连接类型自动切换（MySQL/PostgreSQL/SQLite/...）
- **自动补全**：表名/列名/关键字/SQL 片段/AI 补全
- **多光标**：CodeMirror 原生支持
- **Vim 模式**：@replit/codemirror-vim
- **搜索替换**：CodeMirror Search
- **SQL 格式化**：sql-formatter 集成
- **执行计划可视化**：ExplainPlanViewer
- **自定义主题**：ThemeCustomizerDialog
- **参数提取**：`:param` / `?` / `${var}` 语法
- **拖放表名**：从侧边栏拖入编辑器自动插入

### 9.3 连接对话框（ConnectionDialog.vue, 5980行）

支持 65+ 种数据库类型的连接配置：

- **连接类型选择**：按类别分组（关系型/NoSQL/KV/搜索/向量/消息队列/...）
- **动态表单**：根据 db_type 渲染不同字段
- **SSH 隧道**：TunnelProfileManager
- **连接测试**：实时验证连接可用性
- **密码管理**：系统 Keychain 集成
- **连接导入**：从 Navicat/DBeaver/DataGrip 导入
- **深度链接**：`dbx://connection/new?type=mysql&host=...`

### 9.4 AI 助手（AiAssistant.vue, 2299行）

- **多 Provider**：Claude / OpenAI / Gemini / DeepSeek / Qwen / Ollama / 自定义
- **流式响应**：SSE 流式输出
- **Agent 模式**：多步骤 AI Agent
- **上下文感知**：当前连接/表/SQL 作为上下文
- **SQL 执行**：AI 生成的 SQL 可直接执行
- **对话管理**：保存/加载/删除对话

### 9.5 侧边栏树（TreeItem.vue, 6127行 + ConnectionTree.vue, 1233行）

- **多级树形结构**：连接 → 数据库 → Schema → 表/视图/函数/...
- **懒加载**：展开节点时才加载子节点
- **分页加载**：大量对象分页显示（"加载更多"节点）
- **拖拽排序**：连接/分组拖拽
- **右键菜单**：新建/删除/刷新/复制名称/查看 DDL
- **可见数据库过滤**：选择显示/隐藏特定数据库
- **分组模式**：按对象类型分组（表/视图/函数/存储过程/...）

### 9.6 编辑器设置（EditorSettingsDialog.vue, 4896行）

- 外观（主题/字体/行高）
- 编辑器行为（Tab/缩进/自动补全）

## 14. 关键设计模式

### 14.1 双后端适配

```
前端组件 → api.ts（统一接口）→ tauri.ts（invoke）或 http.ts（fetch）
```

- `api.ts` 通过 `forward()` 实现**懒加载代理**，运行时根据环境选择后端
- `tauri.ts` 和 `http.ts` 导出**完全相同的函数签名**
- 前端组件无需关心运行模式

### 14.2 薄组件 + 厚 lib

```
Vue 组件（UI 渲染）→ Composable（交互逻辑）→ lib/（纯 TS 业务逻辑）
```

- 组件负责模板渲染和事件绑定
- Composable 封装可复用的交互逻辑
- lib/ 包含所有可测试的业务逻辑（125 个测试文件）

### 14.3 异步组件加载

```typescript
// App.vue 中大量使用 defineAsyncComponent 延迟加载
const AiAssistant = defineAsyncComponent(() => import("@/components/editor/AiAssistant.vue"));
const QueryHistory = defineAsyncComponent(() => import("@/components/editor/QueryHistory.vue"));
const DriverStorePage = defineAsyncComponent(() => import("@/components/config/DriverStoreDialog.vue"));
```

减少首屏加载体积，按需加载功能模块。

### 14.4 元数据懒加载 + 缓存

```
侧边栏树展开 → connectionStore.loadXxx()
    → MetadataLoadCoordinator（并发控制）
    → MetadataResultCache（30秒 TTL，160 条上限）
    → MetadataTaskLimiter（限制同时加载任务数）
    → api.listTables() / api.getColumns()
```

### 14.5 Pending Queue 模式（与后端对应）

```typescript
// 前端通过 invoke 获取待处理队列
const pendingSqlFiles = await invoke("pending_open_sql_files");
const pendingDbFiles = await invoke("pending_open_db_files");
const pendingLinks = await invoke("pending_open_connection_links");
```

与 src-tauri 的 Pending Queue 模式对应，实现异步的文件/链接打开。

                                    → dbx-core（Rust）

```

```
Redis PubSub → ws://localhost:4224/api/redis/pubsub/ws?connectionId=xxx

               → Redis PubSub
```

```
实时推送 → EventSource("/api/sse")
           → dbx-web SSE broadcast channels

## 17. 总结

DBX 前端是一个**功能丰富的数据库管理 UI**，其核心价值在于：

1. **双模式运行**：一套代码同时支持 Tauri 桌面版和 Web 版，通过 `api.ts` 懒加载代理无缝切换
2. **65+ 数据库支持**：ConnectionDialog 动态表单适配所有数据库类型
3. **12000+ 行 DataGrid**：类 Excel 的数据浏览/编辑/导出体验
4. **CodeMirror 6 SQL 编辑器**：方言感知、智能补全、Vim 模式、多主题
5. **AI 集成**：多 Provider 流式对话、SQL 生成/执行、Agent 模式
6. **~470 个 API 调用**：覆盖连接/查询/Schema/Redis/Mongo/MQ/Nacos/导出等全功能
7. **厚 lib 薄组件**：466+ 个纯 TS 业务逻辑文件，125 个测试文件，高可测试性
8. **完善的国际化**：7 种语言，按需加载
9. **精细化代码分割**：CodeMirror/ECharts/UI 库独立 chunk，优化首屏加载
10. **现代工具链**：Vite 8 + oxlint + oxfmt + vitest + husky
